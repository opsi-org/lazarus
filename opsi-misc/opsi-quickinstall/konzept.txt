opsi-quick install Konzept

Ziel ist ein einfach herunterzuladendes 'setup programm' 
zur Installation von opsi-server auf unterschiedlichen Distributionen.

Verwendet werden soll dabei das opsi-produkt l-opsi-server

Das ganze soll ein mehrsprachiges Programm werden.
Es soll ein GUI und eine NoGUI (Kommandozeilen) Version geben.

Ablauf der Schritte:

* 1. Begrüßung auf Englisch mit Sprachauswahl
* 2. Begrüßung (lokalisiert) mit Erläuterung etwa:
Will kommen zur Installation des opsi-servers auf diesem Rechner.
Um die restliche Installation automatisch durch führen zu können,
benötigen wir zunächst ein paar Informationen

* Abfrage der einzelnen Informationen und speichern der Informationen
in der Datei l-opsi-server.conf
Die Informationen zu den einzelnen Werten sollen dem control file 
des l-opsi-server Produktes entnommen werden

Beispiel1:
Im control file steht:
-------
[ProductProperty]
type: unicode
name: backend
multivalue: False
editable: False
description: Which backend should be installed ? (mysql needs valid activation file)
values: ["file", "mysql"]
default: ["file"]
--------
Der Name ist hier 'backend'. 
In der description steht die Frage.
Zulässige Werte sind 'file' oder 'mysql'
Der Default ist 'file'

Beispiel2:
Im control file steht:
-------
[ProductProperty]
type: bool
name: update_test
description: Install first from stable and than update to repo_kind
default: False
-----
Der Name ist hier 'update_test'. 
In der description steht die Frage.
Zulässige Werte sind 'true' oder 'false' da der Type bool ist
Der Default ist 'false'

Würde in beiden Beispielen der default gewählt,
so würde in die l-opsi-server.conf geschrieben
-----
backend=file
update_test=false
-----

Besonderheit wäre hier noch: Es soll die zu installierende opsi Version abgefragt werden:
opsi 4.1 oder opsi4.2.
Im Fall von opsi4.1 wäre zusetzen:
opsi_noproxy_online_repository=http://download.opensuse.org/repositories/home:/uibmz:/opsi:/4.1:/
opsi_online_repository=http://download.opensuse.org/repositories/home:/uibmz:/opsi:/4.1:/
Die Urls für 4.2 kenne ich noch nicht.

* Als nächstes soll das angegebene opsi Repository 
in der für die laufende Linux Distribution / Version typischen Form eingetragen werden.
Eine unit dazu baut Jan Werner.

* Nach 'aktivierung des Repos' (z.B. apt update) wird zunächst das Paket opsi-script installiert.
Damit ist unser script-interprete da.

* nun kann das Produkt l-opsi-server ausgeführt werden.

* zum Abschluß gibt es eine Erfolgs / Mißerfolgsmeldung und einen Hinweis auf das Log.
